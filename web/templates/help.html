Help

Uploading Images

Use the Upload page to add images directly through your browser.

  • Select one or more files (JPEG, PNG, WebP, GIF, MP4, WebM).
  • Optionally add tags that will be applied to all uploaded files.
  • Choose a destination folder (e.g. 2026/april) to place the files under; leave blank to save at the gallery root. Missing folders are created automatically.

From the gallery, use Tag all (next to Delete all) to add or remove a list of tags on every image in the current search, or check a few thumbnails and use Tag selected in the batch bar to scope the same dialog to just the selection.

Adding Tags

In the tag input on the image detail page:

  • Type one or more tags separated by spaces and press Add. Each whitespace-separated token becomes its own tag. Wrap a multi-word tag in double quotes to collapse internal spaces to underscores: "red hair" → red_hair. You can also use _ directly.
  • Use category:tagname to add a tag to a specific category.
  • Example: artist:"john doe" blue_eyes 1girl adds "john_doe" as an artist tag and "blue_eyes" and "1girl" as general tags.
  • Autocomplete suggestions appear as you type - use ↑/↓ arrows to navigate, Enter to select.

Tag names

  • Lowercase only: a-z, 0-9, _, (, ), !, @, #, $, ., ~, +, -
  • Maximum 200 characters.

Settings & Maintenance

The Settings page provides:

  • General - toggle the gallery file watcher, set the max file size, and change the number of images per page.
  • Galleries - add, rename, switch between, set default, or delete configured galleries. The Add form takes an optional import file (.db / .json / .zip) so a populated gallery can be created in one step.
  • Authentication - set or remove a password, regenerate the API token.
  • Auto-Tagger - enable one or more taggers (each subfolder of the models directory), set per-tagger confidence thresholds, toggle GPU (CUDA), pick a worker count, and disable or delete a tagger row.
  • Tag actions - add or remove specific tags across every image in the active gallery, run bulk auto-tagging actions (Run untagged / Run all / Remove auto-tagged) per tagger or against every enabled tagger, and bulk-remove all auto-tags, all user tags, or every tag.
  • Maintenance - prune missing images, clean up orphaned thumbnails, rebuild thumbnails, recalculate tag counts, merge auto-tagged-only general tags into their unique categorized counterparts, re-extract metadata, vacuum the database, and list or remove duplicate file paths.
  • Schedule - opt-in daily automation. Pick a time-of-day (default 01:00) and tick any of: Sync gallery, Remove orphaned thumbnails, Run enabled auto-taggers, Recompute tag counts, Merge general tags, Vacuum database. The scheduler iterates every configured gallery in turn at the chosen time and skips silently when another job is already running.
